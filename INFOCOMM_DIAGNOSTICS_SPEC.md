## 2. МОДЕЛЬ ДАННЫХ

### 2.1 Структура вопроса (InfoCommQuestion)

```
┌──────────────────────────────────────────────────────────────────┐
│  InfoCommQuestion                                                │
├──────────────────┬───────────────────────────────────────────────┤
│  id               │ number (1–40)                                │
│  text             │ string (русский)                             │
│  textKz           │ string (казахский)                           │
│  category         │ InfoCommCategory (5 значений)                │
│  questionType     │ "single_choice" | "ranking" | "scenario"     │
│  weight           │ number (1.0 | 1.5 | 2.0)                    │
│  isReversed       │ boolean                                      │
│  difficulty       │ "easy" | "medium" | "hard"                   │
│  competencyTag    │ string (уникальный тег компетенции)          │
│  options          │ { label, labelKz, score }[] (4 варианта)     │
│  scenario?        │ string (только для type=scenario, рус.)      │
│  scenarioKz?      │ string (только для type=scenario, каз.)      │
│  idealOrder?      │ number[] (только для type=ranking)           │
└──────────────────┴───────────────────────────────────────────────┘
```

### 2.2 Категории (InfoCommCategory)

```
┌──────────────────┬────────────────────┬─────────────┬─────────────┐
```

## 5. АЛГОРИТМ СКОРИНГА

### 5.1 Инверсия обратных вопросов

```
applyInfoCommReversal(rawScore, isReversed):
  if isReversed: return 5 - rawScore
  else:          return rawScore

  Пример: raw=1 → adjusted=4, raw=4 → adjusted=1
```

### 5.2 Взвешенный скоринг категории

```
scoreCategoryWeighted(category, answers, lang):

  1. Получить вопросы категории: qs = getInfoCommQuestionsByCategory()[category]

  2. Для каждого вопроса:
     adjusted = applyInfoCommReversal(answers[q.id], q.isReversed)
     questionScore = adjusted × q.weight

  3. weightedSum = Σ(adjusted_i × weight_i)

  4. maxPossible = Σ(4 × weight_i) для всех вопросов категории

  5. rawScore = (weightedSum / maxPossible) × 100

  Пример для motivational (все ответы = 3):
    weightedSum = 3×1.0 + 3×1.0 + 3×1.5 + 3×1.0 + 3×1.5 + (5-3)×1.0 + 3×1.0 + 3×1.5
               = 3 + 3 + 4.5 + 3 + 4.5 + 2 + 3 + 4.5 = 27.5
    maxPossible = 38.0
    rawScore = (27.5 / 38.0) × 100 = 72.4%
```

### 5.4 Общий балл (overallScore)

```
overallScore = Σ(rawScore_i × categoryWeight_i) / Σ(categoryWeight_i)

  categoryWeights:
    motivational:  0.20
    cognitive_info: 0.20
    activity:       0.25  ← наибольший вес
    reflective:     0.15  ← наименьший вес
    outcome:        0.20

  Σ weights = 1.00

  Пример:
    motivational=72, cognitive=65, activity=80, reflective=55, outcome=70
    overall = (72×0.20 + 65×0.20 + 80×0.25 + 55×0.15 + 70×0.20) / 1.00
            = 14.4 + 13.0 + 20.0 + 8.25 + 14.0 = 69.65 → 69.7
```

## 6. СИСТЕМА УВЕРЕННОСТИ (Confidence)

### 6.1 Формула rawConfidence

```
calculateRawConfidence(answers, categories):

  1. completeness = answered / 40

  2. avgVariance = average(categories.map(c → c.internalVariance))
     varianceFactor = min(1, avgVariance / 1.2)

  3. uniqueValues = Set(Object.values(answers)).size
     diversityFactor = min(1, uniqueValues / 4)

  4. confidence = completeness × 0.55
               + varianceFactor × 0.30
               + diversityFactor × 0.15

  5. return min(1, confidence)

  Значения весов:
  ┌────────────────┬───────┬────────────────────────────────────┐
  │  Фактор        │  Вес  │  Описание                          │
  ├────────────────┼───────┼────────────────────────────────────┤
  │  Completeness  │  0.55 │  Доля отвеченных вопросов          │
  │  Variance      │  0.30 │  Разнообразие в категориях         │
  │  Diversity     │  0.15 │  Количество уникальных значений    │
  └────────────────┴───────┴────────────────────────────────────┘
```

### 6.2 Adjusted Confidence

```
adjustedConfidence = rawConfidence × (1 - antiCheat.confidencePenalty)
```

## 11. СХЕМА БАЗЫ ДАННЫХ

### 11.1 Маппинг в таблицу diagnostics_results

```
Таблица: diagnostics_results (существующая, RLS enabled)

Маппинг 5 InfoComm-категорий → 4 DB-колонки:

┌────────────────────────────┬────────────────────────┬───────────────────────┐
│  InfoComm категории        │  DB колонка            │  Формула              │
├────────────────────────────┼────────────────────────┼───────────────────────┤
│  motivational + cognitive  │  cognitive_score       │  round((mot+cog)/2)   │
│  reflective                │  soft_score            │  reflective.rawScore  │
│  activity                  │  professional_score    │  activity.rawScore    │
│  outcome                   │  adaptability_score    │  outcome.rawScore     │
├────────────────────────────┼────────────────────────┼───────────────────────┤
│  average(all 5)            │  average_score         │  round(Σ/5)           │
└────────────────────────────┴────────────────────────┴───────────────────────┘

Идентификация типа теста:
  answers (JSONB) содержит поле: "_test_type": "infocomm"

Пример JSON в answers:
  {
    "1": 3, "2": 4, "3": 2, ... , "40": 3,
    "_test_type": "infocomm"
  }
```

### 11.2 Структура таблицы

```sql
CREATE TABLE diagnostics_results (
  id              UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id         UUID REFERENCES auth.users(id),
  answers         JSONB,            -- {questionId: score, _test_type: "infocomm"}
  cognitive_score INTEGER,           -- (motivational + cognitive_info) / 2
  soft_score      INTEGER,           -- reflective
  professional_score INTEGER,        -- activity
  adaptability_score INTEGER,        -- outcome
  average_score   INTEGER,           -- среднее всех 5 категорий
  test_type       TEXT DEFAULT 'cognitive', -- 'cognitive' | 'infocomm'
  created_at      TIMESTAMPTZ DEFAULT now()
);
```
